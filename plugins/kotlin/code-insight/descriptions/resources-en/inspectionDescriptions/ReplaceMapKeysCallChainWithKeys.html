Reports map { it.key }.toSet() call chains on map receivers that could be replaced with keys.

Note: While this transformation is beneficial for readability and performance, as it avoids unnecessary intermediate collections, it may change call semantics because keys returns a read-only view of the map's keys, whereas the original call chain creates a copy of the keys.

The quick-fix replaces the call chain with the map's keys property.

Example:


  fun test(steps: Map<String, Int>) {
      steps.map { it.key }.toSet()
  }

After the quick-fix is applied:


  fun test(steps: Map<String, Int>) {
      steps.keys
  }
